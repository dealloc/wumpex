    Wumpex

    Documentation for Wumpex.

    Wumpex.Api

    Wraps HTTPoison and configures it according to the Discord specifications.

    Wumpex.Api.Ratelimit

    The Wumpex.Api.Ratelimit module handles dispatching HTTP requests through the Wumpex.Api module, but handles rate limits.

    Wumpex.Api.Ratelimit.StatelessBucket

    Represents a single "bucket" in the rate limit system of Discord as described in the official documentation.

    Wumpex.Api.Ratelimit.Well

    The well handles requests for routes which Wumpex doesn't know the bucket for.

    Wumpex.Base.Ledger

    A facade module for a registry that is either local or distributed. Ledger accomplishes this by using either the built-in Registry module (when compiling for local registry), or :syn (when compiling distributed).

    Wumpex.Base.Websocket

    A behaviour module for implementing the client of a websocket connection.

    Wumpex.Resource.Guild

    Guilds in Discord represent an isolated collection of users and channels, and are often referred to as "servers" in the UI.

    Wumpex.Resource.Guild.ChannelFlags

    Represents system channel flags for a guild.

    Wumpex.Resource.Guild.Member

    Represents a member of a guild.

    Wumpex.Resource.Message

    Represents a message sent in a channel within Discord.

    Wumpex.Resource.Message.Activity

    Sent with Rich Presence-related chat embeds.

    Wumpex.Resource.Message.Application

    Application information sent with Rich presence related embeds.

    Wumpex.Resource.Message.Reaction

    Represents a reaction to a message.

    Wumpex.Resource.Message.Reference

    Reference data sent with crossposted messages.

    Wumpex.Resource.PresenceUpdate

    A user's presence is their current state on a guild. This event is sent when a user's presence or info, such as name or avatar, is updated.

    Wumpex.Resource.Role

    Roles represent a set of permissions attached to a group of users. Roles have unique names, colors, and can be "pinned" to the side bar, causing their members to be listed separately. Roles are unique per guild, and can have separate permission profiles for the global context (guild) and channel context. The @everyone role has the same ID as the guild it belongs to.

    Wumpex.Resource.User

    Users in Discord are generally considered the base entity. Users can spawn across the entire platform, be members of guilds, participate in text and voice chat, and much more. Users are separated by a distinction of "bot" vs "normal". Although they are similar, bot users are automated users that are "owned" by another user. Unlike normal users, bot users do not have a limitation on the number of Guilds they can be a part of.

    Wumpex.Resource.UserFlags

    Struct representing the flags a user account can have.

    Wumpex.Resource.VoiceState

    Represents a user's voice connection status.

    Wumpex.Sharding

    Manages the shards connected to the gateway.

    Wumpex.Sharding.ShardLedger

    Wumpex.Voice

    Contains functionality for connecting and interacting with voice connections.

    Wumpex.Voice.Gateway

    Handles incoming events from the voice gateway. The voice gateway is bound to a specific voice connection of a guild (see Wumpex.Voice.Manager).

    Wumpex.Voice.Manager

    Represents a voice connection and abstracts the Wumpex.Voice.Gateway, Wumpex.Voice.Udp connection and the Wumpex.Voice.Player process.

    Wumpex.Voice.Opcodes

    Provides methods for generating opcodes.

    Wumpex.Voice.Player

    The Player module is responsible for encoding and sending audio data to Discord.

    Wumpex.Voice.Rtp

    Contains methods for generating RTP packets.

    Wumpex.Voice.Udp

    Handles the UDP connection required for voice connections.

    Wumpex.Voice.VoiceLedger

    Keeps track of all voice connections by their name.
