    Wumpex.Gateway.Caching (wumpex v0.1.0) View Source

    Handles state tracking as described in the official Discord documentation.

    This is the second stage in the event processing pipeline, right after Wumpex.Gateway.EventProducer. Once an event is properly handled and the state is up to date, the event will be dispatched to all the listeners.
